# 📄 Day 2 – HTML Forms & Tables
## Overview
This project is part of my MERN Stack learning journey.  
On Day 2, I practiced creating **HTML forms and tables**, focusing on structure, usability, and semantic HTML.  
The project contains **three separate HTML files** demonstrating different concepts: forms, dummy forms, and tables.

## Project Files

| File Name                     | Description |
|-------------------------------|-------------|
| `registration-form.html`      | Event registration form using various input types such as text, email, number, radio buttons, checkboxes, dropdowns, and textarea. |
| `dummy-form.html`             | A sample HTML form with fieldsets, legends, file upload, image button, password input, and a variety of inputs for practice. |
| `tables.html`                 | Demonstrates simple and nested tables, including `rowspan` and `colspan` for complex layouts. |


## What I Learned
- Using **different input types** in forms: text, email, number, password, file, radio, checkbox, and textarea.  
- Grouping form elements using **fieldset** and **legend** for semantic structure.  
- Using **tables** to display data clearly, including **nested tables**.  
- Applying **rowspan** and **colspan** to merge table cells for complex layouts.  
- Basic HTML structure and proper labeling of form inputs for accessibility.  

## Features
- Event Registration Form with multiple input types  
- Dummy form with file upload, image button, and textarea  
- Tables for simple data representation and nested layouts  
- Demonstrates rowspan and colspan usage  
- Semantic grouping of form fields with fieldset and legend

## Key Concepts
- **Forms** collect input from users and are the foundation for interactivity on webpages.  
- **Tables** organize data in a clear and structured manner.  
- **Nested tables** allow hierarchical data representation.  
- **Fieldsets** and **legends** improve semantic grouping and readability of forms.  

## Output
- Functional forms with labels, inputs, checkboxes, radio buttons, dropdowns, file upload, and submit buttons  
- Simple tables displaying structured data  
- Nested tables showing complex tabular layouts  
- Clean, readable HTML with proper indentation
